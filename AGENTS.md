# Repository Instructions

## Editing principles

- Make surgical, narrowly scoped changes. Touch only the code required to complete the requested task.
- Preserve all existing comments unless a comment is demonstrably incorrect because of the requested change. If a comment must change, update it rather than removing it.
- Optimize for human readability. Use clear names, straightforward control flow, and code that is easy to understand and maintain.
- Prefer the simplest solution that fully satisfies the requirements.
- Avoid unnecessary abstractions, architectural changes, speculative flexibility, and premature optimization.
- Follow existing project patterns when they are clear and compatible with these principles.
- Do not perform unrelated cleanup, formatting, refactoring, or dependency changes.
- Do things one at a time.
